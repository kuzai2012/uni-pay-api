## 核心介绍：这是一个什么工具（3分钟）

### 📦 uni-pay-api Skill 是什么？

一句话概括：**一个让 AI 帮你自动接入支付接口的智能助手**

#### 核心定位

| 维度 | 传统方式 | uni-pay-api 方式 |
|------|----------|------------------|
| 接入方式 | 手动复制代码、修改参数 | AI 自动生成、直接写入项目 |
| 支付渠道 | 微信/支付宝/银联分别对接 | 统一接口，一次接入全渠道 |
| 技术栈适配 | 每种语言都要重新写 | 支持 Java/Python/PHP/Go |
| 文档查阅 | 打开无数个文档标签 | 问一句，答案立刻出现 |
| 问题排查 | 响应码手册翻半天 | 实时查询、精准定位 |

#### 三大核心价值

**对产品：**
- 缩短支付功能上线周期（从周级到天级）
- 降低跨部门沟通成本
- 快速验证业务可行性

**对测试：**
- 提供开箱即用的测试工具
- 自动化生成测试参数
- 问题快速定位与反馈

**对开发：**
- 零手工复制代码
- 框架自动识别与适配
- SDK 一次生成、多项目复用

**[重点]** 这不是一个代码模板库，而是一个**智能化的支付接入助手**，能理解你的意图、主动收集参数、自动适配项目结构。

## 核心能力：能帮你做什么（5分钟）

### 🎯 六大能力概览

#### 能力0：快速接入（最核心）

**场景：** "帮我在 Spring Boot 项目里接入微信扫码支付"

**传统流程：**
```
1. 找文档 → 2. 理解接口 → 3. 写 SDK → 4. 写 Controller
→ 5. 配置参数 → 6. 测试调试 → 7. 修复 bug → 8. 上线
预计耗时：3-5 天
```

**uni-pay-api 流程：**
```
1. 告诉 AI 你的需求
2. 填一个结构化表单（签名方式、商户号、密钥等）
3. 等待 30 秒 → 代码已写入项目
预计耗时：10 分钟
```

**[演示] 实际效果展示：**

```bash
# 步骤1：安装 Skill
npx skills add kuzai2012/uni-pay-api --yes

# 步骤2：配置商户信息（最小只需3个字段）
vim ~/.codebuddy/skills/uni-pay-api/config.json
```

```json
{
  "default_merchant_no": "你的商户号",
  "merchant_key": "你的32位MD5密钥",
  "base_url": "https://trade.joinpay.cc"
}
```

```
# 步骤3：对 AI 说一句话
"帮我在 Spring Boot 项目里接入微信扫码支付"

# AI 自动执行：
① 扫描项目 → 识别 Spring Boot 框架
② 弹出表单 → 收集所有必要参数
③ 加载模板 → 替换商户配置
④ 写入文件 → 生成 8 个文件（SDK + Controller + Config）
⑤ 输出报告 → 告诉你下一步该做什么
```

**写入的文件清单：**
```
src/main/java/com/yourcompany/joinpay/
├── sdk/
│   ├── JoinPayClient.java           # HTTP 客户端
│   ├── JoinPaySignature.java        # MD5 签名
│   └── JoinPayRsaSignature.java     # RSA 签名
├── JoinPayConfig.java               # 配置类
├── JoinPayService.java              # 业务服务
├── PayController.java               # 支付接口
└── NotifyController.java            # 回调处理
```

**生成的 REST API：**
```
POST /api/pay/create      # 扫码支付下单
POST /api/pay/wxjsapi     # 微信公众号/小程序支付
POST /api/pay/card        # 付款码支付
GET  /api/pay/query       # 订单查询
POST /api/pay/refund      # 申请退款
GET  /api/pay/refund/query # 退款查询
POST /api/pay/close       # 关闭订单
POST /notify/pay          # 支付回调
POST /notify/refund       # 退款回调
```

AI 会弹出一个结构化表单：

### 🛡️ 环境边界保护

**问题：** AI 写完代码后自动安装依赖，安装失败导致报错，然后 AI 开始乱改接口参数

**解决方案：**

1. **区分错误类型**
   - 环境错误（JDK未安装、依赖缺失）→ 禁止修改接口
   - 代码错误（语法、逻辑）→ 可排查修正

2. **环境安装需用户确认**
   ```
   检测到环境问题：JDK 未安装
   
   建议执行：
   ```bash
   brew install openjdk@11
   ```
   
   是否执行？确认后我将帮您运行。
   ```

3. **严禁修改核心逻辑**
   - 接口路径（如 `/tradeRt/uniPay`）
   - 请求参数名（如 `p0_Version`、`hmac`）
   - 签名算法实现

**[重点] 这保证了 AI 不会因为环境问题而破坏已验证的代码逻辑**
