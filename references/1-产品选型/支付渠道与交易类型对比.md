# 支付渠道与交易类型对比

汇聚支付聚合支付API支持三大主流支付渠道：**微信支付**、**支付宝**、**银联在线支付**。

## 渠道总览对比

| 维度 | 微信支付 | 支付宝 | 银联 |
|------|---------|--------|------|
| **FrpCode前缀** | `WEIXIN_` | `ALIPAY_` | `UNIONPAY_` |
| **用户规模** | 13亿+活跃用户 | 10亿+活跃用户 | 银行卡全覆盖 |
| **适用场景** | 社交/公众号/小程序生态 | 电商/O2O/线下收款 | 大额支付/对公/银行卡 |
| **主扫(扫码)** | ✅ NATIVE | ✅ NATIVE | ✅ NATIVE |
| **被扫(刷卡)** | ✅ CARD | ✅ CARD | ✅ CARD |
| **APP内支付** | ✅ APP / APP3 / APP2 | ✅ APP | ✅ APP |
| **H5网页支付** | ✅ H5 / H5+ | ✅ H5 | ✅ H5（支持ReturnUrl） |
| **公众号支付** | ✅ GZH | ❌ | ❌ |
| **小程序支付** | ✅ XCX / CJXCX(插件) | ❌ | ❌ |
| **服务窗** | ✅ FWC | ✅ FWC | ❌ |
| **收银台** | ✅ SYT | ✅ SYT | ✅ SYT |
| **云微小程序** | ❌ | ❌ | ✅ WXMP |

## 选型决策树

```
开始选型
  │
  ├─ 用户在微信环境内？
  │   │
  │   ├─ 公众号H5页面？ → WEIXIN_GZH（需OpenId + AppId）
  │   │
  │   ├─ 微信小程序？ → WEIXIN_XCX（需OpenId + AppId）
  │   │
  │   └─ 普通H5页面？ → WEIXIN_H5 或 WEIXIN_H5+
  │
  ├─ 用户在APP内？
  │   │
  │   ├─ 原生iOS/Android APP？ → {渠道}_APP
  │   │
  │   └─ 跨平台/H5嵌入APP？ → {渠道}_H5
  │
  ├─ PC端网页？
  │   │
  │   └─ 展示二维码让用户扫 → {渠道}_NATIVE（主扫）
  │
  ├─ 线下实体店收银？
  │   │
  │   └─ 商户扫用户付款码 → {渠道}_CARD（被扫，需AuthCode + 终端信息）
  │
  └─ 需要统一收银入口？
      └─ {渠道}_SYT（收银台，自动识别用户环境）
```

## 各渠道详细 FrpCode 列表

### 微信支付 (WEIXIN)

| FrpCode | 支付方式 | 说明 | 额外必填参数 | 典型场景 |
|---------|---------|------|-------------|---------|
| `WEIXIN_NATIVE` | 主扫(扫码) | 用户扫商户二维码 | q7_AppId(可选，可返回openid) | 线下门店、PC网页 |
| `WEIXIN_CARD` | 被扫(刷卡) | 商户扫用户付款码 | **q6_AuthCode**, **q8_TerminalNo**, **ql_TerminalIp** | 线下POS机、扫码枪 |
| `WEIXIN_APP` | APP支付 | APP内调起微信SDK | **q7_AppId** | 原生APP |
| `WEIXIN_APP3` | APP3支付 | APP跳转小程序 | **q7_AppId**(报备小程序appid) | 通用方案替代原生APP |
| `WEIXIN_H5` | H5支付 | 外部浏览器H5 | - | 手机浏览器H5页面 |
| `WEIXIN_H5_PLUS` | H5增强版 | H5跳转小程序 | **q7_AppId**(报备小程序) | 增强H5体验 |
| `WEIXIN_GZH` | 公众号支付 | 微信公众号JSAPI | **q5_OpenId**, **q7_AppId** | 公众号内H5页面 |
| `WEIXIN_XCX` | 小程序支付 | 小程序内wx.requestPayment | **q5_OpenId**, **q7_AppId** | 微信小程序 |
| `WEIXIN_CJXCX` | 小程序插件 | 小程序插件支付 | - | 第三方小程序插件 |
| `WEIXIN_FWC` | 服务窗 | 微信服务窗支付 | - | 微信服务窗场景 |
| `WEIXIN_SYT` | 收银台 | 统一收银台 | - | 多方式统一入口 |

> **易混淆点辨析**：
> - **GZH vs H5 vs XCX**：GZH必须在公众号内打开；XCX必须在小程序内；H5是外部浏览器
> - **APP3 vs APP+**：APP3是新版通用方案，跳转小程序完成支付
> - **H5 vs H5+**：H5+提供更好的用户体验（跳转小程序），需要报备AppId

### 支付宝 (ALIPAY)

| FrpCode | 支付方式 | 说明 | 额外必填参数 | 典型场景 |
|---------|---------|------|-------------|---------|
| `ALIPAY_NATIVE` | 主扫(扫码) | 用户扫商户二维码 | - | 线下门店、PC网页 |
| `ALIPAY_CARD` | 被扫(刷卡) | 商户扫用户付款码 | **q6_AuthCode**, **q8_TerminalNo**, **ql_TerminalIp** | 线下POS机、扫码枪 |
| `ALIPAY_APP` | APP支付 | APP内调起支付宝SDK | - | 原生APP |
| `ALIPAY_H5` | H5支付 | 手机网页支付 | q9_TransactionModel(MODEL1/2/3可选) | 手机H5页面 |
| `ALIPAY_FWC` | 服务窗 | 支付宝服务窗 | **qb_buyerId**(2088开头) | 支付宝服务窗 |
| `ALIPAY_SYT` | 收银台 | 统一收银台 | - | 多方式统一入口 |

> **H5 TransactionModel 说明**：
> - MODEL1：默认模式，直接跳转支付宝钱包
> - MODEL2：通过中转页再跳转到支付宝
> - MODEL3：返回支付URL自行处理

### 银联 (UNIONPAY)

| FrpCode | 支付方式 | 说明 | 额外必填参数 | 典型场景 |
|---------|---------|------|-------------|---------|
| `UNIONPAY_NATIVE` | 主扫(扫码) | 用户扫商户二维码 | - | 线下门店、PC网页 |
| `UNIONPAY_CARD` | 被扫(刷卡) | 商户扫用户银行卡 | **q6_AuthCode**, **q8_TerminalNo**, **ql_TerminalIp** | 线下POS机 |
| `UNIONPAY_APP` | APP支付 | APP内调起云闪付 | - | 云闪付APP |
| `UNIONPAY_H5` | H5支付 | 手机网页支付 | - | 支持p8_ReturnUrl跳回 |
| `UNIONPAY_SYT` | 收银台 | 银联收银台 | - | 多银行统一入口 |
| `UNIONPAY_WXMP` | 云微小程序 | 云微小程序无感支付 | qm_ContractId(签约ID) | 无感支付场景 |

## 按终端环境推荐

| 终端环境 | 推荐微信 | 推荐支付宝 | 推荐银联 |
|----------|---------|-----------|---------|
| 微信公众号 | ✅ GZH | - | - |
| 微信小程序 | ✅ XCX | - | - |
| 原生iOS/Android APP | ✅ APP | ✅ APP | ✅ APP |
| 手机浏览器(H5) | ✅ H5/H5+ | ✅ H5 | ✅ H5 |
| PC网页 | ✅ NATIVE | ✅ NATIVE | ✅ NATIVE |
| 线下POS/扫码枪 | ✅ CARD | ✅ CARD | ✅ CARD |
| 统一收银入口 | ✅ SYT | ✅ SYT | ✅ SYT |
