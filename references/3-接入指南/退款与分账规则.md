# 退款与分账规则

## 退款接口概览

汇聚支付提供三个退款相关接口：

| 接口 | 路径 | 功能 |
|------|------|------|
| 退款申请 | `/tradeRt/refund` | 发起一笔退款 |
| 退款查询 | `/tradeRt/queryRefund` | 查询单笔退款状态 |
| 退款信息查询 | `/tradeRt/queryRefundInfo` | 查询某支付订单的所有退款记录 |

## 退款申请核心参数

| 参数 | 必填 | 说明 |
|------|------|------|
| p1_MerchantNo | 是 | 商户号 |
| p2_OrderNo | 是 | 原支付订单号 |
| p3_RefundOrderNo | 是 | 退款订单号（唯一） |
| p4_RefundAmount | 是 | 退款金额（元，精确到分） |
| p5_RefundReason | 是 | 退款原因 |
| p6_NotifyUrl | 否 | 退款结果通知地址 |
| hmac | 是 | 签名 |

## 退款规则详解

### 1. 金额规则

```
原订单金额: 100.00 元
    │
    ├─ 第一次退款: 30.00 元  ✓  剩余可退: 70.00
    ├─ 第二次退款: 50.00 元  ✓  剩余可退: 20.00
    ├─ 第三次退款: 20.00 元  ✓  剩余可退: 0.00（全额已退完）
    └─ 第四次退款: 10.00 元  ✗  错误: 退款金额超出可退金额(20090004)
```

- **最小退款金额**: 0.01 元
- **退款金额为0**: 返回 `20090011`
- **退款金额 > 原订单金额**: 返回 `20090003`
- **累计退款 > 原订单金额**: 返回 `20090004`

### 2. 时间规则

| 场景 | 时效 |
|------|------|
| 部分退款 | 支付后 **365天内** 可发起 |
| 全额退款 | 支付后 **365天内** 可发起 |
| 退款到账 | 1~3 个工作日（信用卡可能更长） |

### 3. 状态规则

| 原订单状态 | 能否退款 | 说明 |
|-----------|---------|------|
| SUCCESS（已支付） | ✅ 可以 | 正常情况 |
| NOTPAY（待支付） | ❌ 不行 | 返回 `20090002` 订单未支付 |
| CLOSED（已关闭） | ❌ 不行 | 返回 `20090002` 订单未支付 |
| FULLY_REFUNDED（已全额退） | ❌ 不行 | 返回 `20090012` 已全额退款 |

### 4. 退款订单号规则

- 必须**全局唯一**
- 不能与已有支付订单号或退款订单号重复
- 重复会返回 `20090007` 或 `20090030`
- 建议格式：`REF_{原订单号}_{时间戳}` 或 `RF{yyyyMMddHHmmssSSS}{随机数}`

## 分账退款

如果原订单使用了**实时分账**功能，退款时需要同步处理分账退款：

| 参数 | 必填 | 说明 |
|------|------|------|
| p7_AltRefInfo | 否(分账时必填) | 分账退款信息(JSON) |
| p9_AltOrderNo | 否 | 分账订单号 |
| pa_FundsAccount | 否 | 退款资金账户 |

### 分账退款JSON格式示例

```json
{
  "altRefList": [
    {
      "receiverAccount": "分账接收方账号",
      "refundAmount": "10.00",
      "description": "商品退款"
    }
  ]
}
```

> **注意**：分账退款金额不能超过该接收方的实际分账金额。超出返回 `20090021`。

## 退款查询

两种查询方式的区别：

| 方式 | 接口 | 输入 | 输出 |
|------|------|------|------|
| 按退款单号查 | queryRefund | p2_RefundOrderNo | 单笔退款详情（金额/状态/时间） |
| 按支付单号查 | queryRefundInfo | p2_OrderNo | 该订单所有退款记录列表 |

**推荐**：日常用 `queryRefund` 追踪单笔退款；对账时用 `queryRefundInfo` 检查某订单的全部退款。

## 退款常见响应码

| 响应码 | 含义 | 解决方案 |
|--------|------|---------|
| 100 | 退款成功 | - |
| 20090001 | 原支付订单不存在 | 检查p2_OrderNo是否正确 |
| 20090002 | 原订单未支付 | 订单尚未完成支付，无法退款 |
| 20090003 | 退款金额有误 | 检查金额格式（两位小数） |
| 20090004 | 退款金额超出可退金额 | 已退金额+本次退款 ≤ 原订单金额 |
| 20090005 | 签名验证失败 | 检查签名算法和密钥 |
| 20090007 | 退款订单号重复 | 换一个新的退款订单号 |
| 20090011 | 退款金额为0 | 退款金额必须大于0 |
| 20090012 | 原订单已全额退款 | 无法再次退款 |
| 20090013 | 退款原因不能为空 | 填写退款原因 |
| 20090022 | 原订单无退款信息 | 该订单从未有过退款记录 |
