# 签名与验签规则

汇聚支付支持两种签名方式：**MD5签名** 和 **RSA签名**。商户可在汇聚后台选择并配置。

## 1. MD5 签名算法

适用于大多数中小商户，配置简单。

### 签名流程

```
原始参数Map
    ↓
① 移除 hmac 字段（不参与签名计算）
    ↓
② 按 key 的字母升序排序
    ↓
③ 拼接所有 value（只拼接value，不拼接key名）
    ↓
④ 在末尾追加 merchantKey（32位密钥）
    ↓
⑤ 对整个字符串做 MD5 哈希
    ↓
⑥ 结果转大写 → 即为 hmac 签名值
```

### Java 实现（参考 SignBiz.java）

```java
// 核心逻辑提取自 gw_pay 项目 SignBiz.java:81-113

Properties properties = new Properties();
for (String name : map.keySet()) {
    String value = map.get(name) == null ? "" : map.get(name).toString().trim();
    if ("hmac".equalsIgnoreCase(name)) continue; // ① 跳过hmac
    properties.setProperty(name, value);
}

// ② 按key字母排序
List<String> keys = new ArrayList(properties.keySet());
Collections.sort(keys);

// ③ 拼接所有value
StringBuilder content = new StringBuilder();
for (String key : keys) {
    content.append(properties.getProperty(key));
}

// ④⑤⑥ MD5(拼接值 + 密钥).toUpperCase()
String signStr = content.toString();
String hmac = DigestUtils.md5Hex(signStr + merchantKey).toUpperCase();
```

### Python 实现（参考 uni_pay_client.py）

```python
def sign(params, key):
    filtered = {}
    for k, v in params.items():
        if k.lower() == "hmac": continue  # 跳过hmac
        val = str(v).strip() if v is not None else ""
        if val != "":
            filtered[k] = val

    sorted_keys = sorted(filtered.keys())
    sign_str = "".join(filtered[k] for k in sorted_keys)
    return hashlib.md5((sign_str + key).encode("utf-8")).hexdigest().upper()
```

### Go 实现

```go
func SignMD5(params map[string]string, key string) string {
    var keys []string
    for k := range params {
        if strings.ToLower(k) == "hmac" { continue }
        keys = append(keys, k)
    }
    sort.Strings(keys)

    var buf bytes.Buffer
    for _, k := range keys {
        if v := strings.TrimSpace(params[k]); v != "" {
            buf.WriteString(v)
        }
    }

    hash := md5.Sum([]byte(buf.String() + key))
    return strings.ToUpper(hex.EncodeToString(hash[:]))
}
```

### PHP 实现

```php
function signMD5($params, $key) {
    unset($params['hmac']); // 移除hmac
    ksort($params);         // 按key排序
    
    $signStr = '';
    foreach ($params as $v) {
        $v = trim($v);
        if ($v !== '') {
            $signStr .= $v;
        }
    }
    
    return strtoupper(md5($signStr . $key));
}
```

## 2. RSA 签名算法

适用于安全性要求较高的商户，使用非对称加密。

### 签名流程

```
原始参数Map
    ↓
① 同MD5步骤①~③：移除hmac、按key排序、拼接value
    ↓
② 使用商户 RSA私钥 对 signStr 进行签名（SHA256WithRSA / SHA1WithRSA）
    ↓
③ Base64编码 → 即为 hmac 签名值
```

### 验签流程

```
接收到的参数 + 返回的hmac
    ↓
① 同样移除hmac、排序、拼接value得到 signStr
    ↓
② 使用商户 RSA公钥 验证签名
    ↓
③ 验证通过则数据未被篡改
```

### 注意事项（来自 SignBiz.java）

```java
// 来自 SignBiz.java:127-143 — RSA验签关键实现
else if (encryptType == OrderEncryptTypeEnum.CERT.getValue()) {
    hmac = hmac.replace(" ", "+");  // URL解码处理空格
    MerchantSecret secret = userCache.getMerchantSecretByUserNo(merchantNo);
    String publicKey = secret.getMerchantPublicKey();

    byte[] signData = signStr.getBytes("UTF-8");
    
    // 天威诚信RA证书2048Bit兼容判断
    if (publicKey.length() > 1024) {
        return RSAUtils.verify(signData, publicKey, hmac, "X.509");  // X.509格式
    } else {
        return RSAUtils.verify(signData, publicKey, hmac);            // 默认格式
    }
}
```

**RSA 关键要点**：
1. 私钥用于**签名**（请求时），公钥用于**验签**（验证回调时）
2. 公钥长度 > 1024 时使用 X.509 格式解析
3. Base64解码时注意空格替换为 `+`
4. 推荐使用 **SHA256WithRSA** 算法（比SHA1更安全）

## 3. 签名方式选择指南

| 维度 | MD5签名 | RSA签名 |
|------|--------|--------|
| 安全性 | 中（密钥共享） | 高（非对称加密） |
| 配置复杂度 | 低（仅需32位密钥） | 高（需生成密钥对、上传公钥） |
| 适用场景 | 一般商户 | 金融/高安全需求商户 |
| 性能影响 | 几乎无 | RSA运算略慢 |
| 密钥泄露风险 | 高（泄露即可伪造签名） | 低（仅泄露公钥无法签名） |

## 4. 常见错误排查

| 错误码 | 含义 | 排查方向 |
|--------|------|---------|
| 10080000 | 签名验证失败 | ①检查参与签名的参数是否完整 ②检查key是否正确 ③检查大小写转换 |
| 10080015 | 版本号有误 | 确认p0_Version值是否正确（支付用2.6，退款用2.3） |
| 10080017 | 订单号格式有误 | 只允许英文、数字、下划线，最长50字符 |
| 10080026 | 商户不存在或未激活 | 确认merchant_no是否正确，商户状态是否正常 |

### 签名调试步骤

1. **打印参与签名的参数列表**：确认哪些参数参与了签名
2. **打印排序后的key序列**：确认排序结果是否符合预期
3. **打印拼接后的字符串**：手动复现拼接过程
4. **对比MD5/RSA中间结果**：与服务端日志比对差异
5. **检查编码**：确保统一使用 UTF-8 编码
