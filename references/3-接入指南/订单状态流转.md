# 订单状态流转

## 支付订单状态

汇聚支付订单的状态流转如下：

```
                    ┌──────────────┐
                    │              │
          创建订单   │    NOTPAY    │ ← 初始状态（待支付）
         ─────────→│   (待支付)    │
                    │              │
                    └──────┬───────┘
                           │
              ┌────────────┼────────────────┐
              │            │                │
              ▼            ▼                ▼
     ┌──────────────┐ ┌──────────┐  ┌──────────────┐
     │              │ │          │  │              │
     │   SUCCESS    │ │  CLOSED  │  │    FAILED     │
     │   (已支付)   │ │  (已关闭) │  │   (支付失败)  │
     │              │ │          │  │              │
     └──────┬───────┘ └──────────┘  └──────────────┘
            │
            │ 申请退款
            ▼
     ┌──────────────┐
     │              │
     │   REFUND     │ ← 可部分退款多次
     │   (已退款)   │
     │              │
     └──────────────┘
```

## 状态定义

| 状态 | ra_Status 含义 | 说明 | 是否终态 |
|------|---------------|------|---------|
| NOTPAY / CREATED | 待支付 | 订单创建成功，等待用户支付 | 否 |
| SUCCESS | 已支付 | 支付成功，等待结算 | 否（可退款） |
| CLOSED | 已关闭 | 超时未支付或主动关闭 | 是 |
| FAILED | 支付失败 | 渠道返回失败 | 是 |
| REFUNDING | 退款中 | 退款申请已提交，等待处理 | 否 |
| REFUND | 已退款 | 退款成功（可能部分退款） | 否（可继续退） |
| REFUND_FAIL | 退款失败 | 退款请求被拒绝 | 否（可重新发起） |

## 关闭订单规则

| 条件 | 能否关单 | 说明 |
|------|---------|------|
| 订单状态=NOTPAY | ✅ 可以 | 未支付的超时订单可以关单 |
| 订单状态=SUCCESS | ❌ 不行 | 已支付的订单不能关单，只能退款 |
| 订单状态=CLOSED | ❌ 不行 | 已关闭的订单重复关单会返回错误 |
| 订单状态=FAILED | ❌ 不行 | 失败的订单无需关单 |

> **最佳实践**：建议在下单后 **30分钟** 内轮询订单状态，若仍为 NOTPUT 则调用关单接口释放库存。

## 退款规则

| 规则 | 说明 |
|------|------|
| **支持部分退款** | 单笔订单可多次退款，累计退款金额不超过原订单金额 |
| **退款金额精度** | 精确到分（元），如 10.50 |
| **退款时效** | 通常 1~3 个工作日到账（视银行而定） |
| **全额退款限制** | 全额退款后不可再次退款 |
| **退款顺序** | 先退营销金额（如有分账），再退本金 |
| **退款订单号** | 必须全局唯一，不可重复使用 |

## 幂等性注意事项

| 操作 | 幂等key | 重复提交行为 |
|------|---------|------------|
| 下单 | p2_OrderNo | 返回"10080016 交易已存在" |
| 退款 | p3_RefundOrderNo | 返回"20090007 退款订单号重复"或"20090030 重复退款请求" |
| 关单 | p2_OrderNo | 返回"10083009 关单失败（重复关单）" |

> **建议**：系统层面做好去重，同一订单号不要重复提交。
