## 一、签名相关问题

### Q1: 为什么一直返回签名验证失败(10080000)?

**最常见的原因（按概率排序）：**

1. **包含了 hmac 字段参与签名** — 这是第一大坑！签名时必须排除 hmac
2. **排序方式错了** — 应该按 key 的**字母序**排序，不是自然序，也不是按 value 排序
3. **拼接内容错了** — 只拼接 **value**，不拼接 key，不拼接 `=` 号
4. **忘了追加密钥** — 最后要拼上 `merchantKey` 再做 MD5
5. **大小写问题** — 最终结果要 `.toUpperCase()` 转**大写**
6. **编码问题** — 统一用 UTF-8，不要用 GBK/GB2312
7. **密钥不对** — 确认用的是正确的商户密钥（32位）

**调试技巧**：把参与签名的参数、排序后的keys、拼接的signStr、最终hmac都打出来，逐步对比。

### Q2: MD5和RSA签名如何切换？

在 config.json 中修改 `sign_type` 字段：
- `"sign_type": "MD5"` → 使用 MD5 签名（需要 `merchant_key`）
- `"sign_type": "RSA"` → 使用 RSA 签名（需要 `rsa_private_key` / `rsa_public_key`）

代码层面需要同时支持两种方式，根据商户配置的 `encryptType` 字段动态选择。

### Q3: RSA签名报"公钥格式错误"?

汇聚支持两种公钥格式：
- 普通公钥（≤1024字符）：标准PEM格式
- X.509证书公钥（>1024字符）：需要用 X.509 解析器

如果公钥超过1024字符，务必使用 X.509 方式加载。

## 二、下单相关问题

### Q4: 返回"商户未开通此支付方式"(10080012)?

表示你的商户号还没有申请开通指定的支付产品：

1. 登录 [汇聚商户后台](https://merchant.joinpay.com)
2. 进入「产品管理」或「支付产品」
3. 申请开通需要的支付方式（如微信H5、支付宝APP等）
4. 审核通过后重试

### Q5: Appid未报备(10080029)怎么办?

微信公众号/小程序的 AppId 需要在汇聚后台**报备**：

1. 汇聚商户后台 → 「应用管理」或「AppId管理」
2. 添加你的 AppId（从[微信公众平台](https://open.weixin.qq.com)获取）
3. 等待审核通过
4. 审核通过后使用该 AppId 下单

### Q6: Openid在哪里获取？

**公众号支付(GZH)**:
1. 用户关注公众号后，通过 OAuth 2.0 授权获取
2. 授权URL: `https://open.weixin.qq.com/connect/oauth2/authorize?...`
3. 回调中可拿到 code，再用 code 换取 openid

**小程序支付(XCX)**:
1. 在小程序中调用 `wx.login()` 获取 code
2. 后端用 code 调用微信 `jscode2session` 接口换取 openid
3. 将 openid 传给下单接口

### Q7: 订单号有什么限制？

| 规则 | 限制 |
|------|------|
| 字符集 | 英文字母(a-z, A-Z)、数字(0-9)、下划线(_) |
| 长度 | 最长50字符 |
| 唯一性 | 同一商户下必须唯一 |
| 建议 | 使用 `{日期}{业务前缀}{随机数}` 格式，如 `20260413ORD202604130001` |

### Q8: 被扫(CARD)支付总是失败？

检查以下几点：

1. **付款码有效性** — 付款码有效期通常只有 **1分钟**，过期需用户刷新
2. **终端设备号(q8_TerminalNo)** — 需要在汇聚后台提前报备终端设备
3. **终端IP(ql_TerminalIp)** — 必须是真实的客户端IP，不能用 127.0.0.1
4. **金额限制** — 被扫单笔限额通常较低，确认金额是否超限

## 三、回调相关问题

### Q9: 为什么收不到异步通知？

**排查清单**：

1. [ ] notify_url 是**公网域名/IP**（不能用 localhost / 192.168.x.x / 10.x.x.x）
2. [ ] URL 支持 **POST** 请求（不是 GET）
3. [ ] 服务器防火墙/安全组**放通了80/443端口**
4. [ ] 没有 WAF / CDN 拦截请求
5. [ ] 域名备案正常（国内服务器要求）
6. [ ] SSL 证书有效（HTTPS时）

**临时测试工具**：可以用 ngrok /花生壳 等内网穿透工具暴露本地服务来测试。

### Q10: 回调解签失败？

回调的验签逻辑和请求签名**完全一样**：

1. 取出回调的所有参数
2. 排除 hmac
3. 按 key 排序
4. 拼接 value
5. MD5(拼接值 + key).toUpperCase()
6. 与回调中的 hmac 比较

特别注意：回调参数的类型可能是 `String[]`（多值），取第一个值即可。

### Q11: 回调重复了怎么办？

**这是正常的**！汇聚会发送多达15次重试。

解决方案：**幂等设计**

```java
// 伪代码
String orderNo = params.get("r2_OrderNo");
PaymentOrder order = dao.getByOrderNo(orderNo);

if (order.getStatus() == PAID) {
    return "success"; // 已经处理过了，直接返回成功
}

// 标记为"处理中"，防止并发
if (markProcessing(orderNo)) {
    doBusinessLogic(orderNo);  // 执行业务
    markPaid(orderNo);          // 标记已支付
}
return "success";
```

## 四、退款相关问题

### Q12: 退款多久能到账？

| 支付渠道 | 到账时间 |
|---------|---------|
| 微信支付 | 1~3 个工作日 |
| 支付宝 | 实时~3个工作日（取决于退款来源） |
| 银联 | 1~5 个工作日 |

> 实际到账时间还取决于发卡行的处理速度。

### Q13: 如何查询某订单的所有退款记录？

使用 `queryRefundInfo` 接口，输入原支付订单号(p2_OrderNo)，返回该订单的：

- 原订单金额
- 剩余可退金额
- 所有退款记录列表（每次退款的金额、状态、时间）

### Q14: 退款失败了能重新发起吗？

可以。使用**相同的退款订单号**重新提交即可（汇聚会视为同一笔退款的 retry）。

如果想用新的一笔退款，换一个新的退款订单号。

## 五、其他常见问题

### Q15: 110 系统正忙是什么意思?

汇聚平台瞬时流量过大，暂时无法处理你的请求。**稍后重试即可**，建议间隔 1~3 秒。

如果频繁出现 110，联系汇聚客服确认是否有维护或故障。

### Q16: 限流(10080038)怎么解决?

表示你的商户号在 **1秒内** 请求次数超过了阈值：

1. **降低请求频率** — 加入请求队列/节流
2. **批量处理** — 如果需要批量下单，分散到多个时间段
3. **联系客服提升限额** — 特殊场景可申请提高QPS上限

### Q17: 测试环境和生产环境的区别?

| 维度 | 测试环境 | 生产环境 |
|------|---------|---------|
| base_url | `https://test.joinpay.com` | `https://trade.joinpay.com` |
| 商户号/密钥 | 不同 | 不同 |
| 真实扣款 | 不会真实扣款 | 会真实扣款 |
| 建议测试金额 | 0.01元 | 按实际情况 |

### Q18: 如何区分支付渠道返回的具体错误?

当响应码为 `10083001`（渠道下单失败）时，`rb_CodeMsg` 中会包含渠道的详细信息：

```json
{
  "ra_Code": "10083001",
  "rb_CodeMsg": "渠道下单失败[NO_AUTH]该商户未开通此权限"
}
```

方括号 `[]` 内的就是渠道错误码。常见的：
- **微信**: NO_AUTH / APPID_NOT_BIND / INVALID_REQUEST
- **支付宝**: TRADE_ERROR / ACQ.INVALID_PARAMETER
- **银联**: 系统繁忙 / 交易超时

### Q19: q4_IsShowPic 怎么用?

设置为 `1` 后，微信主扫(NATIVE)的响应中会多返回 `rd_Pic` 字段，内容是 **Base64编码的二维码图片**。

适合场景：需要在前端直接展示二维码图片（而不是让前端自己根据URL生成二维码）时使用。

```html
<!-- 前端使用示例 -->
<img src="data:image/png;base64,{rd_Pic}" />
```

### Q20: H5支付有哪些注意事项?

1. **支付宝H5** 支持 `q9_TransactionModel` 选择跳转模式（MODEL1/2/3）
2. **银联H5** 支持 `p8_ReturnUrl`，支付完成后会跳回这个地址
3. **微信H5** 在普通浏览器中体验有限，建议优先考虑 H5+（跳转小程序）
4. H5支付需要配置**合法的支付域名**（微信/支付宝后台都需要配置）
