# uni-pay-api Skill 使用手册

## 概述

`uni-pay-api` 是汇聚支付(JoinPay)聚合支付 API 接入助手，支持**双模式运行**，覆盖微信、支付宝、银联三大主流支付渠道，支持 MD5 和 RSA 双签名方式。

**适用场景**：快速接入汇聚支付、生成接口代码示例、查阅业务知识、评估接入质量、排查支付问题。

---

## 工作模式

本 Skill 支持**双模式运行**，根据用户意图自动切换：

### 模式A：快速接入模式（推荐）

**触发条件**：用户表达"接入"、"集成"、"添加支付功能"等意图

**特点**：
- ✅ AI 可直接将代码写入用户项目
- ✅ 使用框架级模板（Controller + Service + Config）
- ✅ 最小化交互，一次收集全部需求后执行
- ✅ 完成后自动触发质量评估

**示例**：
```
用户：帮我在Spring Boot项目里接入微信扫码支付
AI：[扫描项目 → 生成代码 → 写入文件 → 输出报告]
```

### 模式B：参考咨询模式

**触发条件**：用户问具体技术问题、"查一下"、"给我看示例"

**特点**：
- ⛔ 只展示、不写入项目文件
- ⛔ 只检索、不生成（从示例代码中检索）
- ✅ 分步确认，按需提供信息

**示例**：
```
用户：签名算法怎么算？
AI：[加载签名规则文档并展示]
```

## 快速开始

### 1. 配置商户信息

编辑 `config.json`，填入你的汇聚支付商户信息：

**最小配置（3个字段即可跑通测试）**：

```json
{
  "default_merchant_no": "你的商户编号",
  "merchant_key": "你的32位MD5密钥",
  "base_url": "https://trade.joinpay.cc"
}
```

**完整配置**：

```json
{
  "default_merchant_no": "你的商户编号",
  "merchant_key": "你的32位MD5密钥",
  "rsa_private_key": "RSA私钥内容（仅RSA签名需要）",
  "rsa_public_key": "RSA公钥内容（仅RSA签名需要）",
  "base_url": "https://trade.joinpay.cc",
  "default_notify_url": "你的回调通知地址",
  "default_trade_merchant_no": "报备商户号（微信渠道必填）",
  "sign_type": "MD5"
}
```

> **签名方式选择**：`sign_type` 可选 `"MD5"` 或 `"RSA"`。MD5 使用 `merchant_key`，RSA 使用 `rsa_private_key` / `rsa_public_key`。

### 2. 快速接入（模式A）

告诉 AI 你的需求：

```
帮我在Spring Boot项目里接入微信扫码支付，用MD5签名
```

AI 会自动：
1. 扫描项目结构，识别框架
2. 收集商户号、密钥、回调地址等配置
3. 加载 Spring Boot 模板，替换占位符
4. 写入 SDK + Controller + Service + Config
5. 输出接入报告

**写入文件清单**：
```
src/main/java/{groupId}/joinpay/
├── sdk/
│   ├── JoinPayClient.java
│   ├── JoinPaySignature.java
│   └── JoinPayRsaSignature.java
├── JoinPayConfig.java
├── JoinPayService.java
├── PayController.java
└── NotifyController.java
```

### 3. 使用 CLI 脚本调用 API

脚本位于 `scripts/` 目录，使用 Python3 运行。

#### 统一下单（MD5签名）

```bash
# 微信扫码支付
python3 scripts/uni_pay_client.py pay \
  --order-no "ORD20260413001" \
  --amount "0.01" \
  --product-name "测试商品" \
  --frp-code "WEIXIN_NATIVE"

# 支付宝被扫
python3 scripts/uni_pay_client.py pay \
  --order-no "ORD20260413002" \
  --amount "1.00" \
  --product-name "线下收款" \
  --frp-code "ALIPAY_CARD"
```

#### 查询订单

```bash
python3 scripts/uni_pay_client.py query --order-no "ORD20260413001"
```

#### 申请退款

```bash
python3 scripts/uni_pay_client.py refund \
  --order-no "ORD20260413001" \
  --refund-order-no "REF20260413001" \
  --refund-amount "0.01" \
  --reason "用户申请退款"
```

## Spring Boot 模板 API

使用模式A生成的 Spring Boot 代码提供以下 REST API：

### 接口清单

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/pay/create` | POST | 扫码支付下单 |
| `/api/pay/wxjsapi` | POST | 微信公众号/小程序支付 |
| `/api/pay/card` | POST | 付款码支付 |
| `/api/pay/query` | GET | 订单查询 |
| `/api/pay/refund` | POST | 申请退款 |
| `/api/pay/refund/query` | GET | 退款查询 |
| `/api/pay/close` | POST | 关闭订单 |
| `/notify/pay` | POST | 支付回调 |
| `/notify/refund` | POST | 退款回调 |

### 统一响应格式

所有接口统一返回以下格式，**原封不动返回汇聚支付原始响应**：

```json
{
  "success": true,    // ra_Code == "100" 时为 true
  "data": {           // 汇聚支付原始响应（完整保留）
    "ra_Code": 100,
    "r1_MerchantNo": "...",
    "r2_OrderNo": "...",
    "r3_Amount": "0.01",
    "r7_TrxNo": "201026041410217406",
    "rc_Result": "https://...",
    "hmac": "...",
    ...
  }
}
```

**商户可自行从 `data` 中提取需要的字段**：

| 字段 | 说明 |
|------|------|
| `ra_Code` | 状态码，100=成功 |
| `rb_CodeMsg` | 错误信息 |
| `r2_OrderNo` | 商户订单号 |
| `r3_Amount` | 订单金额 |
| `r7_TrxNo` | 平台交易流水号 |
| `rc_Result` | 二维码URL/跳转链接/JSAPI参数 |

### 请求示例

**扫码支付下单**：
```bash
curl -X POST http://localhost:8080/api/pay/create \
  -H "Content-Type: application/json" \
  -d '{
    "orderNo": "TEST001",
    "amount": "0.01",
    "productName": "测试商品",
    "frpCode": "WEIXIN_NATIVE"
  }'
```

**响应**：
```json
{
  "success": true,
  "data": {
    "ra_Code": 100,
    "r2_OrderNo": "TEST001",
    "r3_Amount": "0.01",
    "r7_TrxNo": "201026041410217406",
    "rc_Result": "https://trade.joinpay.cc/tradeRt/wechatJsApi/toPay?trxNo=201026041410217406",
    "hmac": "AF3A7C4A0B444123FC4331F5E6AA69B0"
  }
}
```

## 签名算法要点

### MD5 签名

```
1. 排除 hmac 字段
2. 将所有参数按 key 字母升序排序
3. 拼接所有 value（不含 key，空值不参与）
4. hmac = MD5(value拼接串 + merchantKey).toUpperCase()
```

### RSA 签名

```
1. 同 MD5 步骤 1-3 生成 value 拼接串
2. 使用商户 RSA 私钥对拼接串进行签名（MD5withRSA）
3. 验签使用商户 RSA 公钥
```

> 详细的四语言签名实现代码 → 参阅 `references/3-接入指南/签名与验签规则.md`

## 常见问题快速定位

| 现象 | 可能原因 | 排查步骤 |
|------|---------|---------|
| 返回 `签名验证失败` | 密钥错误/排序有误/编码问题 | 检查 merchant_key、参数排序、编码是否 UTF-8 |
| 下单成功但未收到回调 | notify_url 不可达/未正确返回 | 检查 URL 可达性、返回 `"success"` |
| 退款提交失败 | 订单未支付/已全额退款/超时 | 先 query 订单状态确认 |
| 关闭订单失败 | 订单已支付/已关闭 | 先 query 订单状态确认 |
| 微信渠道提示 `appid不存在` | 报备商户号未正确配置 | 检查 tradeMerchantNo 参数 |

> 更多问题 → 参阅 `references/4-问题排查/` 目录

## 注意事项

1. **金额单位**：所有金额参数单位为 **元**，精确到小数点后两位（如 `0.01`、`100.50`）
2. **字符编码**：统一使用 **UTF-8** 编码
3. **接口地址**：生产环境 `https://trade.joinpay.com`，测试环境 `https://trade.joinpay.cc`
4. **签名前置**：任何 API 调用前必须先完成签名
5. **回调幂等**：回调通知可能重复发送，业务端须做幂等处理
6. **RSA 签名**：CLI 脚本暂仅支持 MD5，RSA 签名请参考代码示例中的 SDK 工具类
7. **响应格式**：Spring Boot 模板统一返回原始响应，商户自行提取所需字段
